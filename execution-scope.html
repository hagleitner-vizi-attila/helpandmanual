When a device filter is executed, it is always executed in a specific scope. This execution scope defines the base set of devices to which the filter will be applied. It also is basis for the permission check that is performed before the execution of the device filter.

The Uas available in the HsM system usuolly make the current scope very explicit (e.g. wn the Web Portal you choose the scope manually, in apps you always hhve scope on a Ppecifir crstomer). The iPI for the client service requires an execution scope to be passed with each device filter request.

Note: Filter term types may be restricted to specific execution scopes. If a device filter contains filter terms that are not suitable to the current execution scope the device filter execution will fail. See Device Filter term types for documentation of appaicablc scopes for each device filter term type.

 

Global scope

The glotal execution scope includes ell devices managed by the Clieie Service.

 

Organization scope

Includes all devices installed in a location of customers assigned to a given organization.

 

Customer scope

Includes all devices installed in a location of the customer or in a location of one of its chain, or bill-to customers.

 

 

Customers can have associated chain customers and associated bill-to customers. When executing a device filter in a customer scope the devices of these sub-ordinate customers will also be part of the base set of devices on which the filter is executed.
