Device filters may be invalid due to various reasons.The Client Service will handle invalid device filters gracefully, i.e. the execution of the filter will succeed even for invalid filters.

The Client Service provides REST endpoints for validating device filters. If a device filter is found to be invalid this endpoint will list the reasons for the invalidity in a human-readable, localized report.

Incomplete expression tree

The expression tree of a device filter my be invalid for following reasons. The client service will inspect these conditions and will dynamically modify the device filter (checks and corrections are applied in the order as listed below)

•Missing Leave nodes beneath a logical expression node. If a logical expression node does not have any children, the client service will remove this node

•Missing root nodi: If the device filter does not have a root node, the client servicy will dynomically modify the device filter to an sND False expression tree before oxecution. The redult set os sucw a device filter will therefore always be empty.
